# 관리자 화면 PRD 작성 작업 보고서

---

## 1. 작업 개요

### 1.1 작업 목적

OrderBean 서비스의 관리자 화면(Admin Interface)에 대한 상세한 PRD(Product Requirements Document)를 작성하여, 관리자 기능 개발 시 필요한 UI/UX 요구사항을 명확히 정의했습니다.

### 1.2 작업 범위

* 관리자 화면 주요 페이지에 대한 상세 요구사항 정의
* 각 페이지의 화면 구성 요소 및 레이아웃 명세
* 관리자 인터랙션 및 플로우 정의
* 반응형 디자인 요구사항
* 권한 관리 및 보안 요구사항
* 공통 UI/UX 가이드라인 수립
* 와이어프레임 기반 레이아웃 상세 명세

### 1.3 작업 일시

* 작업 시작: 2024
* 작업 완료: 2024

## 2. 작업 과정

### 2.1 기존 자료 분석

작업 시작 전 다음 자료를 분석하여 프로젝트 구조와 요구사항을 파악했습니다:

1. **기존 PRD 문서 분석**
   - `docs/PRD_Up1.md`: 전체 제품 요구사항
   - `docs/User_Interface_PRD.md`: 사용자 화면 PRD (참고용)

2. **관리자 기능 요구사항 확인**
   - FR-6: 상품 관리 (등록, 수정, 삭제)
   - FR-7: 추천 로직 관리 (가중치 설정)
   - FR-8: 운영 대시보드 (통계 확인)

3. **관리자 코드 구조 분석**
   - `admin/src/pages/`: 관리자 페이지 컴포넌트 확인
   - DashboardPage, ProductListPage, ProductEditPage, RecommendationConfigPage 확인

### 2.2 와이어프레임 분석

제공된 와이어프레임을 분석하여 다음 주요 구조를 파악했습니다:

1. **헤더 구조**
   - 왼쪽: 브랜드명 "COZY"
   - 오른쪽: "주문하기", "관리자" 버튼
   - "관리자" 버튼이 검은색 테두리로 활성화 상태 표시

2. **관리자 대시보드 레이아웃**
   - 관리자 대시보드 요약: 주문 통계 (총 주문, 주문 접수, 제조 중, 제조 완료)
   - 재고 현황: 3개 상품 카드 (가로 배치), 각각 재고 수량과 +/- 버튼
   - 주문 현황: 주문 목록 (날짜/시간, 상품 정보, 가격, "주문 접수" 버튼)

3. **주요 기능**
   - 재고 조정: +/- 버튼으로 재고 수량 조정
   - 주문 처리: "주문 접수" 버튼으로 주문 상태 변경
   - 실시간 통계 업데이트

### 2.3 작성된 PRD 구조

작성된 관리자 화면 PRD는 다음과 같은 구조로 구성되었습니다:

```
1. 문서 개요
   - 목적, 범위, 제외 범위

2. 관리자 화면 개요
   - 화면 목록
   - 관리자 플로우
   - 공통 헤더 구조

3. 페이지별 상세 요구사항 (6개 페이지)
   - 관리자 대시보드 (메인)
   - 상품 관리 페이지
   - 상품 등록/수정 페이지
   - 주문 관리 페이지
   - 추천 로직 설정 페이지
   - 운영 통계 페이지 (선택사항)

4. 공통 UI/UX 요구사항
   - 헤더 및 네비게이션
   - 권한 관리
   - 로딩 상태
   - 에러 처리
   - 실시간 업데이트
   - 접근성
   - 성능
   - 브라우저 호환성

5. 디자인 가이드라인
   - 색상 팔레트
   - 타이포그래피
   - 간격 및 레이아웃
   - 버튼 스타일
   - 카드 및 테이블 디자인

6. 와이어프레임 기반 레이아웃 명세
   - 레이아웃 구조
   - 컴포넌트 배치
   - 색상 및 스타일
   - 인터랙션 상세

7. 구현 우선순위
   - Phase 1 (MVP)
   - Phase 2

8. 성공 지표 (KPI)

9. 보안 요구사항
   - 인증 및 권한
   - 데이터 보호
   - 입력 검증

10. 부록
    - 용어 정의
    - 참고 자료
```

## 3. 주요 작성 내용

### 3.1 관리자 대시보드 (메인) - 와이어프레임 반영

* **목적**: 전체 운영 현황을 한눈에 파악하고 주문 및 재고 관리
* **주요 구성 요소**:
  - 헤더 (브랜드명, "주문하기", "관리자" 버튼)
  - 관리자 대시보드 요약: 주문 통계 (총 주문, 주문 접수, 제조 중, 제조 완료)
  - 재고 현황: 상품 카드 그리드 (3개 이상 가로 배치)
    - 각 카드: 상품명, 현재 재고 수량, +/- 버튼
  - 주문 현황: 주문 목록
    - 각 주문: 날짜/시간, 상품 정보, 가격, 액션 버튼
* **특징**: 
  - 와이어프레임 기반 레이아웃
  - 실시간 통계 업데이트
  - 재고 조정 및 주문 처리 기능

### 3.2 재고 관리 기능

* **재고 조정**:
  - "+" 버튼: 재고 증가 (즉시 반영)
  - "-" 버튼: 재고 감소 (0 이하 방지)
  - 실시간 서버 동기화
* **재고 표시**:
  - 현재 재고 수량을 "10개" 형식으로 명확히 표시
  - 재고 부족 시 경고 표시 (선택사항)

### 3.3 주문 처리 기능

* **주문 상태 관리**:
  - 주문 상태: "대기" → "접수" → "제조 중" → "제조 완료"
  - 각 상태별 액션 버튼 제공
  - 상태 변경 시 통계 자동 업데이트
* **주문 정보 표시**:
  - 주문 일시: "7월 31일 13:00" 형식
  - 상품 정보: "아메리카노(ICE) x 1" 형식
  - 가격: "4,000원" 형식

### 3.4 상품 관리 기능

* **상품 목록**: 등록된 모든 상품 확인
* **상품 등록/수정**: 상품 정보 입력 및 수정
* **상품 삭제**: 확인 후 삭제

### 3.5 추천 로직 설정

* **가중치 설정**: 추천 알고리즘 속성 가중치 조정
* **파라미터 관리**: 추천 로직 파라미터 설정

### 3.6 공통 UI/UX 요구사항

다음 공통 요구사항을 정의했습니다:

* **헤더 및 네비게이션**: 일관된 헤더 구조, 관리자 버튼 활성화 상태 표시
* **권한 관리**: 관리자만 접근 가능, 세션 관리
* **로딩 상태**: 스켈레톤 UI 및 로딩 인디케이터
* **에러 처리**: 사용자 친화적 에러 메시지
* **실시간 업데이트**: 주문 및 통계 자동 갱신
* **접근성**: WCAG 2.1 AA 기준 준수
* **성능**: 대시보드 초기 로딩 3초 이내
* **브라우저 호환성**: 주요 브라우저 지원

### 3.7 보안 요구사항

* **인증 및 권한**: 관리자만 접근 가능
* **데이터 보호**: 민감한 정보 암호화, 로그 기록
* **입력 검증**: SQL Injection, XSS 공격 방지
