## 주요 내용

**목적**: 프론트엔드 TDD RED 단계 테스트 보고서와 백엔드 테스트 실행 보고서를 분석해, 구현해야 할 기능을 기능 요구사항과 비기능 요구사항으로 분류한 문서입니다.

**주요 구성**:

1. **백엔드 구현 요구사항** (섹션 1)
   - 기능 요구사항 7개 (추천 상품 조회, 설문 데이터 조회, 추천 알고리즘 등)
   - 비기능 요구사항 4개 (성능 최적화, 캐싱, 코드 품질 등)

2. **프론트엔드 구현 요구사항** (섹션 2)
   - 기능 요구사항 5개 (API 모킹, 에러 처리, 엣지 케이스 등)
   - 비기능 요구사항 4개 (테스트 독립성, 테스트 환경 개선 등)

3. **통합 구현 요구사항** (섹션 3)
   - API 통신 프로토콜 정의, 데이터 형식 검증 등

4. **우선순위 요약 및 구현 로드맵** (섹션 4-5)
   - 우선순위별 분류 (🔴 높음 / 🟡 중간 / 🟢 낮음)
   - 4단계 구현 로드맵 제안

**결론**: 총 **23개의 구현 요구사항**을 식별했으며, 그중 **6개의 핵심 기능**이 즉시 구현 필요로 분류되어 있습니다.

# OrderBean 관리자 화면 PRD (Admin Interface Product Requirements Document)

---

## 1. 문서 개요

### 1.1 목적

본 문서는 OrderBean 서비스의 관리자 화면(Admin Interface)에 대한 상세 요구사항을 정의합니다. 관리자가 주문 관리, 재고 관리, 상품 관리, 운영 통계 확인 등을 수행할 수 있는 UI/UX 요구사항을 포함합니다.

### 1.2 범위

* 관리자 화면 주요 페이지에 대한 상세 요구사항
* 각 페이지의 화면 구성 요소 및 레이아웃
* 관리자 인터랙션 및 플로우
* 반응형 디자인 요구사항
* 권한 관리 및 보안 요구사항
* UI/UX 가이드라인

### 1.3 제외 범위

* 사용자 화면 (별도 문서: `docs/Frontend_UI_PRD_Customer.md`)
* 백엔드 API 상세 명세
* 데이터베이스 스키마
* 인증/인가 시스템 상세 구현

---

## 2. 관리자 화면 개요

### 2.1 화면 목록

OrderBean 관리자 화면은 다음 주요 페이지로 구성됩니다:

1. **관리자 대시보드 (AdminDashboardPage)** - `/admin` (메인 대시보드)
2. **상품 관리 페이지 (ProductManagementPage)** - `/admin/products`
3. **상품 등록/수정 페이지 (ProductEditPage)** - `/admin/products/new`, `/admin/products/:id/edit`
4. **주문 관리 페이지 (OrderManagementPage)** - `/admin/orders`
5. **추천 로직 설정 페이지 (RecommendationConfigPage)** - `/admin/recommendation-config`
6. **운영 통계 페이지 (AnalyticsPage)** - `/admin/analytics` (선택사항)

### 2.2 관리자 플로우

```
관리자 대시보드 (메인)
  ├─ 주문 현황 확인 및 처리
  ├─ 재고 현황 확인 및 조정
  ├─ 상품 관리
  ├─ 추천 로직 설정
  └─ 운영 통계 확인
```

### 2.3 공통 헤더 구조

모든 관리자 페이지에서 공통으로 사용되는 헤더:

* **왼쪽**: 브랜드 로고/이름 ("COZY" 또는 "OrderBean")
* **오른쪽**: 
  * "주문하기" 버튼 (사용자 주문 페이지로 이동)
  * "관리자" 버튼 (관리자 페이지로 이동, 현재 활성화 상태 표시)

---

## 3. 페이지별 상세 요구사항

### 3.1 관리자 대시보드 (AdminDashboardPage) - 메인 대시보드

#### 3.1.1 목적

* 전체 운영 현황을 한눈에 파악
* 주문 현황 실시간 모니터링 및 처리
* 재고 현황 확인 및 빠른 조정
* 주요 지표 요약 표시

#### 3.1.2 화면 구성 요소

**헤더 영역 (공통)**
* 왼쪽: 브랜드 로고/이름 ("COZY" 또는 "OrderBean")
* 오른쪽: 
  * "주문하기" 버튼 (사용자 주문 페이지로 이동)
  * "관리자" 버튼 (현재 활성화 상태, 검은색 테두리로 표시)

**관리자 대시보드 요약 영역**
* 섹션 타이틀: "관리자 대시보드"
* 주문 통계 요약:
  * 형식: "총 주문 {total} / 주문 접수 {received} / 제조 중 {processing} / 제조 완료 {completed}"
  * 예시: "총 주문 1 / 주문 접수 1 / 제조 중 0 / 제조 완료 0"
  * 각 통계는 실시간으로 업데이트
  * 통계 항목:
    * **총 주문**: 전체 주문 수
    * **주문 접수**: 접수된 주문 수
    * **제조 중**: 현재 제조 중인 주문 수
    * **제조 완료**: 완료된 주문 수

**재고 현황 영역**
* 섹션 타이틀: "재고 현황"
* 상품 카드 그리드 레이아웃 (가로 배치, 3열 이상)
* 각 상품 카드 구성 요소:
  * **상품명**: 
    * 예: "아메리카노(ICE)", "아메리카노(HOT)", "카페라떼"
  * **현재 재고 수량**: 
    * 형식: "{quantity}개"
    * 예: "10개"
    * 숫자로 명확히 표시
  * **재고 조정 버튼**: 
    * "+" 버튼: 재고 증가 (클릭 시 +1)
    * "-" 버튼: 재고 감소 (클릭 시 -1)
    * 버튼은 상품명 아래에 배치
    * 재고가 0일 때 "-" 버튼 비활성화 또는 경고

**주문 현황 영역**
* 섹션 타이틀: "주문 현황"
* 주문 목록 (최신순 정렬):
  * 각 주문 항목 구성:
    * **주문 일시**: 
      * 형식: "{월}월 {일}일 {시간}:{분}"
      * 예: "7월 31일 13:00"
    * **주문 상품 정보**: 
      * 형식: "{상품명} x {수량}"
      * 예: "아메리카노(ICE) x 1"
      * 커스터마이징 옵션이 있는 경우 함께 표시
      * 예: "아메리카노(ICE) (샷 추가) x 1"
    * **주문 금액**: 
      * 형식: "{가격}원"
      * 예: "4,000원"
      * 통화 형식으로 표시
    * **주문 상태**: 
      * 상태: "대기", "접수", "제조 중", "제조 완료", "취소"
      * 상태별 색상 구분 (선택사항)
    * **액션 버튼**: 
      * "주문 접수" 버튼 (주문 상태가 "대기"일 때 표시)
      * "제조 시작" 버튼 (주문 상태가 "접수"일 때 표시)
      * "제조 완료" 버튼 (주문 상태가 "제조 중"일 때 표시)
      * 버튼은 주문 항목 오른쪽에 배치

#### 3.1.3 인터랙션 요구사항

**재고 조정**
* "+" 버튼 클릭 시:
  * 재고 수량 즉시 +1 증가
  * 변경사항 서버에 저장
  * 성공 시 시각적 피드백 제공
* "-" 버튼 클릭 시:
  * 재고가 0보다 클 때만 감소
  * 재고가 0일 때 경고 메시지 또는 버튼 비활성화
  * 재고 수량 즉시 -1 감소
  * 변경사항 서버에 저장

**주문 처리**
* "주문 접수" 버튼 클릭 시:
  * 주문 상태를 "대기" → "접수"로 변경
  * 주문 통계 요약 자동 업데이트
  * 버튼이 "제조 시작"으로 변경
* "제조 시작" 버튼 클릭 시:
  * 주문 상태를 "접수" → "제조 중"으로 변경
  * 주문 통계 요약 자동 업데이트
  * 버튼이 "제조 완료"로 변경
* "제조 완료" 버튼 클릭 시:
  * 주문 상태를 "제조 중" → "제조 완료"로 변경
  * 주문 통계 요약 자동 업데이트
  * 주문 항목을 완료 섹션으로 이동하거나 표시 변경

**실시간 업데이트**
* 주문 통계는 새 주문이 들어올 때 자동 업데이트
* 주문 목록은 새 주문이 추가될 때 자동 갱신
* 재고 변경 시 즉시 반영

#### 3.1.4 데이터 구조

**주문 통계 데이터**
```typescript
{
  total: number;        // 총 주문 수
  received: number;     // 주문 접수 수
  processing: number;   // 제조 중 수
  completed: number;    // 제조 완료 수
}
```

**재고 데이터**
```typescript
{
  productId: string;
  productName: string;
  stock: number;       // 현재 재고 수량
}
```

**주문 데이터**
```typescript
{
  orderId: string;
  orderDate: Date;      // 주문 일시
  items: Array<{
    productName: string;
    quantity: number;
    options?: string[]; // 커스터마이징 옵션
    price: number;
  }>;
  totalPrice: number;
  status: 'pending' | 'received' | 'processing' | 'completed' | 'cancelled';
}
```

#### 3.1.5 반응형 디자인

* **데스크톱**: 
  * 재고 현황 카드 3열 이상 가로 배치
  * 주문 현황 목록 전체 너비 사용
* **태블릿**: 
  * 재고 현황 카드 2열 배치
  * 주문 현황 목록 세로 스크롤
* **모바일**: 
  * 재고 현황 카드 1열 배치
  * 주문 현황 목록 세로 스크롤
  * 버튼 크기 터치하기 쉽게 조정

**상품 정보 입력 영역**

  * 상품명 (필수)

## 4. 공통 UI/UX 요구사항

### 4.1 헤더 및 네비게이션

* 모든 관리자 페이지에서 일관된 헤더 구조 유지
* 헤더 구성:
  * 왼쪽: 브랜드 로고/이름 (클릭 시 관리자 대시보드로 이동)
  * 오른쪽: "주문하기" 버튼, "관리자" 버튼
* "관리자" 버튼은 관리자 페이지에서 활성화 상태 표시 (검은색 테두리)

### 4.2 권한 관리

* 관리자 권한이 있는 사용자만 접근 가능
* 권한이 없는 사용자가 접근 시 에러 메시지 표시 및 리다이렉트
* 세션 만료 시 자동 로그아웃 및 로그인 페이지로 이동

### 4.3 로딩 상태

* API 호출 중 로딩 인디케이터 표시
* 데이터 로딩 중 스켈레톤 UI 표시
* 주문 목록 등 대량 데이터 로딩 시 페이지네이션 또는 무한 스크롤

### 4.4 에러 처리

* 네트워크 에러 시 사용자 친화적 에러 메시지
* 권한 오류 시 명확한 안내 메시지
* 폼 검증 에러는 해당 필드 근처에 표시
* 서버 에러 시 재시도 옵션 제공

### 4.5 실시간 업데이트

* 주문 목록은 새 주문이 들어올 때 자동 갱신 (폴링 또는 WebSocket)
* 주문 통계는 상태 변경 시 자동 업데이트
* 재고 변경 시 즉시 반영

### 4.6 접근성 (Accessibility)

* 키보드 네비게이션 지원
* 스크린 리더를 위한 ARIA 레이블 제공
* 색상 대비 비율 WCAG 2.1 AA 기준 준수
* 포커스 인디케이터 명확히 표시

### 4.7 성능 요구사항

* 대시보드 초기 로딩 시간 3초 이내
* 주문 목록 스크롤 시 부드러운 성능
* 재고 조정 시 즉각적인 피드백

### 4.8 브라우저 호환성

* 최신 버전 Chrome, Firefox, Safari, Edge 지원
* 태블릿 환경에서도 주요 기능 사용 가능

* 주요 액션: 큰 크기, 강조 색상
* 보조 액션: 중간 크기, 보조 색상
* 위험 액션 (삭제 등): 빨간색 또는 경고 색상
* 버튼 상태: 활성, 비활성, 로딩 상태 구분

### 5.5 카드 및 테이블 디자인

* 카드: 흰색 배경, 얇은 테두리 또는 그림자 효과

* 주요 지표 요약 표시

#### 3.1.2 화면 구성 요소

**헤더 영역 (공통)**
* 왼쪽: 브랜드 로고/이름 ("COZY" 또는 "OrderBean")
* 오른쪽: 
  * "주문하기" 버튼 (사용자 주문 페이지로 이동)
  * "관리자" 버튼 (현재 활성화 상태, 검은색 테두리로 표시)

**관리자 대시보드 요약 영역**
* 섹션 타이틀: "관리자 대시보드"
* 주문 통계 요약:
  * 형식: "총 주문 {total} / 주문 접수 {received} / 제조 중 {processing} / 제조 완료 {completed}"
  * 예시: "총 주문 1 / 주문 접수 1 / 제조 중 0 / 제조 완료 0"
  * 각 통계는 실시간으로 업데이트
  * 통계 항목:
    * **총 주문**: 전체 주문 수
    * **주문 접수**: 접수된 주문 수
    * **제조 중**: 현재 제조 중인 주문 수
    * **제조 완료**: 완료된 주문 수

**재고 현황 영역**
* 섹션 타이틀: "재고 현황"
* 상품 카드 그리드 레이아웃 (가로 배치, 3열 이상)
* 각 상품 카드 구성 요소:
  * **상품명**: 
    * 예: "아메리카노(ICE)", "아메리카노(HOT)", "카페라떼"
  * **현재 재고 수량**: 
    * 형식: "{quantity}개"
    * 예: "10개"
    * 숫자로 명확히 표시
  * **재고 조정 버튼**: 
    * "+" 버튼: 재고 증가 (클릭 시 +1)
    * "-" 버튼: 재고 감소 (클릭 시 -1)
    * 버튼은 상품명 아래에 배치
    * 재고가 0일 때 "-" 버튼 비활성화 또는 경고

**주문 현황 영역**
* 섹션 타이틀: "주문 현황"
* 주문 목록 (최신순 정렬):
  * 각 주문 항목 구성:
    * **주문 일시**: 
      * 형식: "{월}월 {일}일 {시간}:{분}"
      * 예: "7월 31일 13:00"
    * **주문 상품 정보**: 
      * 형식: "{상품명} x {수량}"
      * 예: "아메리카노(ICE) x 1"
      * 커스터마이징 옵션이 있는 경우 함께 표시
      * 예: "아메리카노(ICE) (샷 추가) x 1"
    * **주문 금액**: 
      * 형식: "{가격}원"
      * 예: "4,000원"
      * 통화 형식으로 표시
    * **주문 상태**: 
      * 상태: "대기", "접수", "제조 중", "제조 완료", "취소"
      * 상태별 색상 구분 (선택사항)
    * **액션 버튼**: 
      * "주문 접수" 버튼 (주문 상태가 "대기"일 때 표시)
      * "제조 시작" 버튼 (주문 상태가 "접수"일 때 표시)
      * "제조 완료" 버튼 (주문 상태가 "제조 중"일 때 표시)
      * 버튼은 주문 항목 오른쪽에 배치

#### 3.1.3 인터랙션 요구사항

**재고 조정**
* "+" 버튼 클릭 시:
  * 재고 수량 즉시 +1 증가
  * 변경사항 서버에 저장
  * 성공 시 시각적 피드백 제공
* "-" 버튼 클릭 시:
  * 재고가 0보다 클 때만 감소
  * 재고가 0일 때 경고 메시지 또는 버튼 비활성화
  * 재고 수량 즉시 -1 감소
  * 변경사항 서버에 저장

**주문 처리**
* "주문 접수" 버튼 클릭 시:
  * 주문 상태를 "대기" → "접수"로 변경
  * 주문 통계 요약 자동 업데이트
  * 버튼이 "제조 시작"으로 변경
* "제조 시작" 버튼 클릭 시:
  * 주문 상태를 "접수" → "제조 중"으로 변경
  * 주문 통계 요약 자동 업데이트
  * 버튼이 "제조 완료"로 변경
* "제조 완료" 버튼 클릭 시:
  * 주문 상태를 "제조 중" → "제조 완료"로 변경
  * 주문 통계 요약 자동 업데이트
  * 주문 항목을 완료 섹션으로 이동하거나 표시 변경

**실시간 업데이트**
* 주문 통계는 새 주문이 들어올 때 자동 업데이트
* 주문 목록은 새 주문이 추가될 때 자동 갱신

#### 6.2.4 인터랙션 상세
