# OrderBean 사용자 화면 PRD (User Interface Product Requirements Document)

---

## 1. 문서 개요

### 1.1 목적

본 문서는 OrderBean 서비스의 사용자 화면(User Interface)에 대한 상세 요구사항을 정의합니다. 사용자가 커피 취향 설문부터 주문, 피드백까지의 전체 여정을 지원하는 UI/UX 요구사항을 포함합니다.

### 1.2 범위

* 사용자 화면 6개 페이지에 대한 상세 요구사항
* 각 페이지의 화면 구성 요소 및 레이아웃
* 사용자 인터랙션 및 플로우
* 반응형 디자인 요구사항
* UI/UX 가이드라인

### 1.3 제외 범위

* 관리자 화면 (별도 문서로 작성 예정)
* 백엔드 API 상세 명세
* 데이터베이스 스키마

---

## 2. 사용자 화면 개요

### 2.1 화면 목록

OrderBean 사용자 화면은 다음 6개의 주요 페이지로 구성됩니다:

1. **홈 페이지 (HomePage)** - `/`
2. **설문 페이지 (SurveyPage)** - `/survey`
3. **추천 결과 페이지 (RecommendationPage)** - `/recommendations`
4. **상품 주문 페이지 (ProductOrderPage)** - `/order` (메인 주문 화면)
5. **주문 확인 페이지 (OrderConfirmPage)** - `/order/confirm`
6. **피드백 페이지 (FeedbackPage)** - `/feedback`

### 2.2 사용자 플로우

```
홈 페이지
  ↓ (취향 설문 시작)
설문 페이지
  ↓ (설문 완료)
추천 결과 페이지
  ↓ (상품 선택 또는 직접 주문)
상품 주문 페이지 (메인 주문 화면)
  ↓ (장바구니에 담기)
장바구니 (하단 고정)
  ↓ (주문하기)
주문 확인 페이지
  ↓ (주문 완료)
피드백 페이지
```

### 2.3 공통 헤더 구조

모든 페이지에서 공통으로 사용되는 헤더:

* **왼쪽**: 브랜드 로고/이름 ("COZY" 또는 "OrderBean")
* **오른쪽**: 
  * "주문하기" 버튼 (장바구니로 이동 또는 주문 페이지로 이동)
  * "관리자" 버튼 (관리자 페이지로 이동, 권한 필요)

## 3. 페이지별 상세 요구사항

### 3.1 홈 페이지 (HomePage)

#### 3.1.1 목적

* OrderBean 서비스 소개 및 가치 제안 전달
* 사용자의 문제점 인식 및 해결책 제시
* 취향 설문 시작 유도

#### 3.1.2 화면 구성 요소

**Hero Section (상단 영역)**
* 브랜드명: "OrderBean"
* 메인 타이틀: "나에게 딱 맞는 커피를 찾아보세요"
* 서브 타이틀: "복잡한 원두 선택, 이제 그만! 간단한 설문으로 당신만의 완벽한 커피를 추천해드립니다."
* CTA 버튼: "취향 설문 시작하기" (클릭 시 `/survey`로 이동)
* 보조 텍스트: "3분이면 끝! 전문 용어 없이 쉽게"
* 커피 원두 일러스트레이션

**Problem Section (문제 제시 영역)**
* 섹션 타이틀: "이런 고민 있으신가요?"
* 문제점 3가지 카드 형태로 표시:
  1. "원두 종류가 너무 많아요" - 수백 가지 원두 중 선택의 어려움
  2. "전문 용어가 어려워요" - 산미, 바디감 등 전문 용어의 진입 장벽
  3. "실패 경험이 두려워요" - 잘못된 선택에 대한 두려움

**Solution Section (해결책 제시 영역)**
* 섹션 타이틀: "OrderBean이 해결해드립니다"
* 해결책 3가지:
  1. "간단한 설문" - 5문항 이내의 쉬운 질문
  2. "맞춤 추천" - 취향에 맞는 커피 3~5종 추천
  3. "실패 없는 선택" - 추천 이유를 쉽게 설명

**How It Works Section (사용 방법 안내 영역)**
* 섹션 타이틀: "이렇게 시작하세요"
* 3단계 프로세스:
  1. 취향 설문 (📝 아이콘)
  2. 맞춤 추천 (☕ 아이콘)
  3. 주문하기 (🛒 아이콘)
* 단계 간 화살표(→)로 연결

**CTA Section (최종 행동 유도 영역)**
* 타이틀: "지금 바로 시작해보세요"
* 설명: "전문 지식 없이도 나에게 맞는 커피를 찾을 수 있습니다"
* CTA 버튼: "무료로 취향 설문 시작하기" (클릭 시 `/survey`로 이동)

#### 3.1.3 인터랙션 요구사항

* 모든 CTA 버튼은 호버 시 시각적 피드백 제공
* 버튼 클릭 시 부드러운 페이지 전환 애니메이션
* 스크롤 시 섹션별 페이드인 효과 (선택사항)

#### 3.1.4 반응형 디자인

* 데스크톱: 전체 레이아웃 유지, 가로 배치
* 태블릿: 카드 레이아웃 2열 배치
* 모바일: 카드 레이아웃 1열 배치, 텍스트 크기 조정

### 3.2 설문 페이지 (SurveyPage)

#### 3.2.1 목적

* 사용자의 커피 취향을 파악하기 위한 설문 수집
* 간단하고 직관적인 질문으로 사용자 부담 최소화
* 설문 완료 후 추천 결과로 자동 이동

#### 3.2.2 화면 구성 요소

**헤더 영역**
* 페이지 타이틀: "커피 취향 설문"
* 진행률 표시 바 (Progress Bar)
* 설문 문항 수 표시 (예: "1/5")

**설문 문항 영역**
* 현재 문항 번호 및 질문 텍스트
* 답변 옵션 (라디오 버튼, 체크박스, 슬라이더 등 문항 유형에 따라)
* 이전/다음 버튼
* 건너뛰기 옵션 (선택사항)

**설문 문항 목록 (5문항 이내)**

1. **산미 선호도**
   * 질문: "어떤 산미를 선호하시나요?"
   * 답변 옵션: "강한 산미", "적당한 산미", "약한 산미", "산미 없음"
   * 입력 방식: 라디오 버튼

2. **쓴맛 선호도**
   * 질문: "쓴맛은 어떤 정도를 선호하시나요?"
   * 답변 옵션: "강한 쓴맛", "적당한 쓴맛", "약한 쓴맛", "쓴맛 없음"
   * 입력 방식: 라디오 버튼

3. **고소함 선호도**
   * 질문: "고소한 맛을 선호하시나요?"
   * 답변 옵션: "매우 선호", "선호", "보통", "비선호"
   * 입력 방식: 라디오 버튼

4. **우유 사용 여부**
   * 질문: "우유를 넣어 드시나요?"
   * 답변 옵션: "항상 넣음", "가끔 넣음", "넣지 않음"
   * 입력 방식: 라디오 버튼

5. **커피 경험 수준** (선택사항)
   * 질문: "커피를 얼마나 드시나요?"
   * 답변 옵션: "매일", "주 3-4회", "주 1-2회", "가끔"
   * 입력 방식: 라디오 버튼

**하단 영역**
* "이전" 버튼 (첫 문항에서는 비활성화)
* "다음" 버튼 (마지막 문항에서는 "완료" 버튼으로 변경)
* 진행률 표시

#### 3.2.3 인터랙션 요구사항

* 답변 선택 시 즉시 시각적 피드백 제공
* 필수 문항 미입력 시 "다음" 버튼 비활성화 또는 경고 메시지
* 설문 완료 시 로딩 인디케이터 표시 후 `/recommendations`로 이동
* 답변 변경 시 진행률 자동 업데이트

#### 3.2.4 데이터 검증

* 모든 필수 문항은 반드시 입력되어야 함
* 설문 제출 전 최종 확인 (선택사항)

#### 3.2.5 반응형 디자인

* 모바일: 한 번에 하나의 문항만 표시, 스와이프 제스처 지원
* 데스크톱: 문항과 답변 옵션을 한 화면에 표시

### 3.3 추천 결과 페이지 (RecommendationPage)

#### 3.3.1 목적

* 설문 결과를 기반으로 추천된 커피 상품 3~5종 표시
* 각 추천 상품에 대한 추천 사유 제공
* 상품 선택 및 주문으로 이어지는 플로우 제공

#### 3.3.2 화면 구성 요소

**헤더 영역**
* 페이지 타이틀: "맞춤 커피 추천"
* 부제목: "당신의 취향에 맞는 커피를 찾았어요"
* 설문 다시하기 링크 (선택사항)

**추천 결과 영역**
* 추천 상품 카드 리스트 (3~5개)
* 각 카드 구성 요소:
  * 상품 이미지 (또는 플레이스홀더)
  * 상품명
  * 가격 정보
  * 추천 사유 텍스트 (예: "산미를 선호하시는 분께 추천합니다")
  * 맛 속성 태그 (산미, 쓴맛, 고소함 등)
  * "상세보기" 버튼
  * "주문하기" 버튼

**추천 상품 카드 레이아웃**
* 카드 형태의 그리드 레이아웃
* 각 카드는 클릭 가능하며 상세 페이지로 이동
* 호버 시 카드 확대 효과

**하단 영역**
* "다른 추천 받기" 버튼 (설문 다시하기)
* "전체 상품 보기" 링크 (선택사항)

#### 3.3.3 인터랙션 요구사항

* 페이지 로드 시 추천 결과 페이드인 애니메이션
* 상품 카드 클릭 시 `/products/:id`로 이동
* "주문하기" 버튼 클릭 시 주문 페이지로 이동 (상품 정보 포함)
* 로딩 중에는 스켈레톤 UI 표시

#### 3.3.4 데이터 표시

* 추천 상품은 추천 점수 순으로 정렬
* 각 상품의 추천 사유는 사용자가 이해하기 쉬운 언어로 작성
* 가격 정보는 통화 형식으로 표시

#### 3.3.5 반응형 디자인

* 데스크톱: 3열 그리드 레이아웃
* 태블릿: 2열 그리드 레이아웃
* 모바일: 1열 레이아웃, 카드 전체 너비 사용

### 3.4 상품 주문 페이지 (ProductOrderPage) - 메인 주문 화면

#### 3.4.1 목적

* 커피 상품 목록을 표시하고 직접 선택하여 주문할 수 있는 메인 주문 화면
* 각 상품에 대한 커스터마이징 옵션 제공
* 장바구니에 상품 추가 및 주문 진행

#### 3.4.2 화면 구성 요소

**헤더 영역 (공통)**
* 왼쪽: 브랜드 로고/이름 ("COZY" 또는 "OrderBean")
* 오른쪽: 
  * "주문하기" 버튼 (장바구니로 이동)
  * "관리자" 버튼 (관리자 페이지로 이동)

**상품 메뉴 영역 (메인 콘텐츠)**
* 상품 카드 그리드 레이아웃 (가로 배치, 3열 이상)
* 각 상품 카드 구성 요소:
  * **상품 이미지**: 
    * 이미지 플레이스홀더 (대각선 X 표시)
    * 실제 상품 이미지가 로드되면 표시
  * **상품명**: 
    * 예: "아메리카노(ICE)", "아메리카노(HOT)", "카페라떼"
  * **기본 가격**: 
    * 통화 형식으로 표시 (예: "4,000원", "5,000원")
  * **간단한 설명**: 
    * 플레이스홀더: "간단한 설명..."
    * 실제 상품 설명 텍스트
  * **커스터마이징 옵션**: 
    * 체크박스 형태의 옵션들
    * 예시 옵션:
      * "□ 샷 추가 (+500원)" - 체크 시 가격에 500원 추가
      * "□ 시럽 추가 (+0원)" - 무료 옵션
    * 옵션 선택 시 실시간 가격 업데이트
  * **"담기" 버튼**: 
    * 상품을 장바구니에 추가하는 버튼
    * 클릭 시 선택한 옵션과 함께 장바구니에 추가

**장바구니 영역 (하단 고정)**
* 섹션 타이틀: "장바구니"
* 장바구니 아이템 목록:
  * 각 아이템 표시 형식:
    * "상품명 (옵션명) X 수량"
    * 예: "아메리카노(ICE) (샷 추가) X 1"
    * 예: "아메리카노(HOT) X 2"
  * 각 아이템의 가격 표시
  * 아이템 삭제 버튼 (선택사항)
* 총 금액:
  * "총 금액" 레이블
  * 금액을 굵게 표시 (예: "12,500원")
  * 모든 아이템의 합계 자동 계산
* "주문하기" 버튼:
  * 장바구니에 아이템이 있을 때 활성화
  * 클릭 시 주문 확인 페이지로 이동

#### 3.4.3 인터랙션 요구사항

* 커스터마이징 옵션 체크/해제 시:
  * 해당 상품 카드의 가격 실시간 업데이트 (선택사항)
  * "담기" 버튼 클릭 시 선택한 옵션이 장바구니에 반영
* "담기" 버튼 클릭 시:
  * 장바구니에 상품 추가
  * 장바구니 영역에 새 아이템 표시
  * 총 금액 자동 업데이트
  * 시각적 피드백 제공 (예: 토스트 메시지)
* 장바구니 아이템 수량 변경:
  * +/- 버튼 또는 직접 입력 (선택사항)
  * 수량 변경 시 총 금액 자동 업데이트
* 장바구니가 비어있을 때:
  * "주문하기" 버튼 비활성화 또는 숨김
  * 안내 메시지 표시 (선택사항)

#### 3.4.4 데이터 구조

**상품 카드 데이터**
```typescript
{
  id: string;
  name: string;
  basePrice: number;
  description: string;
  imageUrl?: string;
  customizationOptions: Array<{
    id: string;
    name: string;
    price: number; // 추가 가격 (0원일 수도 있음)
  }>;
}
```

**장바구니 아이템 데이터**
```typescript
{
  productId: string;
  productName: string;
  basePrice: number;
  selectedOptions: Array<{
    optionId: string;
    optionName: string;
    optionPrice: number;
  }>;
  quantity: number;
  totalPrice: number; // (basePrice + sum of options) * quantity
}
```

#### 3.4.5 반응형 디자인

* **데스크톱**: 
  * 상품 카드 3열 이상 가로 배치
  * 장바구니 하단 고정 또는 사이드바 형태
* **태블릿**: 
  * 상품 카드 2열 배치
  * 장바구니 하단 고정
* **모바일**: 
  * 상품 카드 1열 배치
  * 장바구니 하단 고정 (스크롤 가능)
  * 장바구니 접기/펼치기 기능 (선택사항)

* "주문 내역 보기" 링크 (선택사항)

* 주문 완료 시 주문 번호 복사 기능
* 주문 실패 시 에러 메시지 표시

#### 3.5.4 데이터 검증

* 배송 정보 필수 항목 검증

### 3.6 피드백 페이지 (FeedbackPage)

#### 3.6.2 화면 구성 요소

  * 상품 이미지

**만족도 평가 영역**

  * 시각적 표시: ★☆☆☆☆ (클릭 가능)
* 세부 평가 항목 (선택사항):
  * 맛: ★☆☆☆☆
  * 향: ★☆☆☆☆
  * 가격 대비 만족도: ★☆☆☆☆

**추가 의견 영역** (선택사항)
* 자유 텍스트 입력:
  * 텍스트 영역 (최대 500자)
  * 플레이스홀더: "추가 의견을 남겨주세요"

**액션 버튼 영역**
* "건너뛰기" 버튼 (선택사항)

**제출 완료 영역** (평가 완료 시)

* "홈으로 돌아가기" 버튼

## 4. 공통 UI/UX 요구사항

### 4.1 헤더 및 네비게이션

* 모든 페이지에서 일관된 헤더 구조 유지
* 헤더 구성:
  * 왼쪽: 브랜드 로고/이름 (클릭 시 홈으로 이동)
  * 오른쪽: "주문하기" 버튼, "관리자" 버튼
* "주문하기" 버튼:
  * 장바구니가 비어있을 때: 주문 페이지로 이동
  * 장바구니에 아이템이 있을 때: 장바구니 표시 또는 주문 확인 페이지로 이동
* 모바일 환경에서 햄버거 메뉴 제공 (필요한 경우)

### 4.2 로딩 상태

* API 호출 중 로딩 인디케이터 표시
* 스켈레톤 UI를 활용한 콘텐츠 로딩 표시
* 로딩 시간이 3초 이상일 경우 진행률 표시

### 4.3 에러 처리

* 네트워크 에러 시 사용자 친화적 에러 메시지
* 404 에러 페이지 제공
* 폼 검증 에러는 해당 필드 근처에 표시

### 4.4 접근성 (Accessibility)

* 키보드 네비게이션 지원
* 스크린 리더를 위한 ARIA 레이블 제공
* 색상 대비 비율 WCAG 2.1 AA 기준 준수
* 포커스 인디케이터 명확히 표시

### 4.5 성능 요구사항

* 초기 페이지 로딩 시간 3초 이내
* 이미지 지연 로딩 (Lazy Loading)
* 코드 스플리팅을 통한 번들 크기 최적화

### 4.6 브라우저 호환성

* 최신 버전 Chrome, Firefox, Safari, Edge 지원
* 모바일 브라우저 (iOS Safari, Chrome Mobile) 지원

## 6. 와이어프레임 기반 레이아웃 명세

### 6.1 와이어프레임 분석 결과

제공된 와이어프레임을 기반으로 다음과 같은 레이아웃 구조를 반영했습니다:

**메인 주문 페이지 (ProductOrderPage) 레이아웃:**
* 상단: 헤더 (브랜드명 "COZY", "주문하기", "관리자" 버튼)
* 중앙: 상품 메뉴 영역 (3개 이상의 상품 카드 가로 배치)
* 하단: 장바구니 영역 (고정, 스크롤 가능)

**상품 카드 구조:**
* 이미지 영역 (플레이스홀더 또는 실제 이미지)
* 상품명
* 기본 가격
* 간단한 설명
* 커스터마이징 옵션 (체크박스)
* "담기" 버튼

**장바구니 구조:**
* "장바구니" 타이틀
* 아이템 목록 (상품명, 옵션, 수량, 가격)
* 총 금액
* "주문하기" 버튼

### 6.2 와이어프레임 기반 상세 명세

#### 6.2.1 레이아웃 구조

* **헤더 높이**: 고정 높이 (약 60-80px)
* **상품 카드 크기**: 
  * 가로: 동일한 너비 (그리드 레이아웃)
  * 세로: 콘텐츠에 따라 자동 조정
* **장바구니 높이**: 
  * 최소 높이: 콘텐츠에 따라 조정
  * 최대 높이: 화면의 30-40% (스크롤 가능)

#### 6.2.2 컴포넌트 배치

* 상품 카드는 가로로 3개 이상 배치 (데스크톱)
* 각 카드 간 간격: 16px~24px
* 장바구니는 화면 하단에 고정
* 장바구니 내부 아이템 간 간격: 8px~12px

#### 6.2.3 색상 및 스타일

* 헤더 배경: 밝은 색상 또는 흰색
* 상품 카드 배경: 흰색 또는 밝은 배경
* 카드 테두리: 얇은 테두리 또는 그림자 효과
* 장바구니 배경: 흰색 또는 약간 어두운 배경으로 구분
* 버튼: 강조 색상 사용

#### 6.2.4 인터랙션 상세

* 상품 카드 호버 시: 약간의 확대 효과 또는 그림자 증가
* "담기" 버튼 클릭 시: 버튼 색상 변경 또는 로딩 인디케이터
* 장바구니 아이템 추가 시: 부드러운 애니메이션으로 표시
* 총 금액 업데이트: 숫자 변경 애니메이션 (선택사항)
